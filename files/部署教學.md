# 📦 庫存管理 App — 完整部署教學

**適合對象：** 完全不熟悉程式的使用者  
**完成時間：** 約 30–45 分鐘  
**費用：** 完全免費（Supabase Free + Vercel Free）

---

## 總覽：三個步驟

```
① Supabase 建資料庫  →  ② Vercel 部署網站  →  ③ 新增團隊帳號
```

---

## 步驟一：建立 Supabase 資料庫（約 10 分鐘）

### 1-1 建立帳號
1. 前往 https://supabase.com 點「Start your project」
2. 用 GitHub 帳號登入（沒有就先註冊 GitHub）
3. 點「New project」
4. 填入：
   - **Organization**：選你的帳號
   - **Name**：`inventory-app`
   - **Database Password**：設定一個強密碼（記下來）
   - **Region**：選 `Northeast Asia (Tokyo)`（台灣最近）
5. 點「Create new project」，等待約 1 分鐘

### 1-2 執行資料庫腳本
1. 在左側選單點 **SQL Editor**
2. 點右上角「New query」
3. 複製 `supabase_setup.sql` 檔案的**全部內容**貼上
4. 點「Run」（或按 Ctrl+Enter）
5. 看到 `Success. No rows returned` 表示成功

### 1-3 取得 API 金鑰
1. 左側選單點 **Project Settings** → **API**
2. 記下這兩個值（等下要用）：
   - **Project URL**：`https://xxxxxx.supabase.co`
   - **anon public key**：`eyJhbGci...`（很長的一串）

---

## 步驟二：部署到 Vercel（約 10 分鐘）

### 2-1 上傳程式碼到 GitHub
1. 前往 https://github.com 登入（沒有就先註冊）
2. 點右上角「+」→「New repository」
3. Repository name：`inventory-app`
4. 選「Private」（私密），點「Create repository」
5. 將整個 `inventory-app` 資料夾上傳：
   - 點「uploading an existing file」
   - 把資料夾裡的**所有檔案**拖進去
   - 點「Commit changes」

> 💡 如果會用 Git，可以用 `git push` 上傳

### 2-2 在 Vercel 部署
1. 前往 https://vercel.com 點「Sign Up」
2. 選「Continue with GitHub」登入
3. 點「Add New Project」→ Import `inventory-app`
4. **重要：設定環境變數** — 展開「Environment Variables」
   - 新增第一個：
     - Name：`VITE_SUPABASE_URL`
     - Value：貼上步驟 1-3 的 **Project URL**
   - 新增第二個：
     - Name：`VITE_SUPABASE_ANON_KEY`
     - Value：貼上步驟 1-3 的 **anon public key**
5. 點「Deploy」，等待約 2 分鐘
6. 部署完成後，Vercel 會給你一個網址，例如：
   `https://inventory-app-xxx.vercel.app`

### 2-3 設定 Supabase 允許你的網域
1. 回到 Supabase → **Project Settings** → **Authentication**
2. 在「Redirect URLs」加入你的 Vercel 網址
3. 在「Site URL」也填入 Vercel 網址
4. 點「Save」

---

## 步驟三：新增團隊成員帳號（約 5 分鐘）

### 3-1 建立帳號
1. 回到 Supabase → **Authentication** → **Users**
2. 點「Invite user」
3. 輸入成員的電子郵件，點「Send invitation」
4. 成員會收到邀請信，點擊連結設定密碼

### 3-2 設定權限角色

在 Supabase SQL Editor 執行以下指令（每個成員各執行一次）：

```sql
-- 將 user@email.com 改成成員的電子郵件
-- 將 editor 改成對應角色（admin / editor / viewer）

update profiles
set role = 'editor', name = '成員姓名'
where id = (
  select id from auth.users where email = 'user@email.com'
);
```

### 角色說明

| 角色 | 可以做什麼 |
|------|-----------|
| **admin（管理員）** | 全部功能，包含刪除商品、設定匯率 |
| **editor（編輯）** | 新增/編輯庫存、新增訂單、登記付款 |
| **viewer（檢視者）** | 只能查看，不能修改任何資料 |

## 步驟四：安裝到手機桌面

### iPhone（Safari 必須用 Safari）
1. 用 Safari 開啟 Vercel 網址
2. 點下方工具列中間的「分享」按鈕（方塊+箭頭圖示）
3. 選「加入主畫面」
4. 點「新增」
5. 桌面就會出現 App 圖示！

### Mac（Chrome 或 Safari）
1. 用 Chrome 開啟網址
2. 網址列右側會有「安裝」圖示（電腦+箭頭）
3. 點擊後選「安裝」
4. App 會出現在 Applications 資料夾

## 常見問題

**Q：登入後顯示「找不到資料」？**  
A：確認環境變數有正確設定，可以在 Vercel 的 Settings → Environment Variables 確認。

**Q：Vercel 部署失敗？**  
A：確認所有檔案都已上傳，特別是 `package.json` 和 `vite.config.js`。

**Q：成員收不到邀請信？**  
A：請檢查垃圾信件夾。或在 Supabase → Authentication → Users 直接設定密碼。

**Q：想換自己的網域名稱？**  
A：在 Vercel → Settings → Domains 可以綁定自訂網域。

**Q：資料安全嗎？**  
A：所有資料儲存在 Supabase 的安全雲端，有 Row Level Security 保護。Vercel 使用 HTTPS 加密傳輸。
